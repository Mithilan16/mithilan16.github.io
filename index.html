Operation Station for Bugleboard Robot

This project is one my favourites. I developed a multi-threaded client application that would serve as operation station for tele-operated robot. I designed the client side application layer protocol after reviewing the application layer protocol for the robot. This application was responsible for sending commands to the robot and in return receiving telemetry data over a reliable TCP/IP connection. I would say the most enjoyable part of this project was packaging and interpreting raw data at the byte level. This project was a great learning experience on good data communcation practices and transfering data over TCP/IP socket connections.

ARAIG Flight Simulation Shell

ARAIG (As Real As It Gets) is a commercial multi-sensory suit intended to enhance virtual reality experiences with force and audio feedback making you feel as if you were actually there! ARAIG has many applications that range from gaming and movies to training for various dangerous or strenuous environments.

Dr.Coleshill, our 3rd semester object-oriented software development professor, had been working on a cross-disciplinary applied research project between Seneca's School of Aviation and School of Information Communications Technology. The goal of this project? To enhance flight stimulation training. In light of this, we were given the opportunity to write ARAIG evaluation control software, as a final project. We were tasked with creating a C++ based object oriented console application that emulated loading and executing flight stimulation data on the suit. The project involved the use of important C++ concepts such as inheritance, polymorphism, and heavy use of the STL library (unique/shared pointers, vectors, lists). This was a great learning experience which allowed us to use knowledge learned in the class room to devise, design and implement software that has real world implications.
